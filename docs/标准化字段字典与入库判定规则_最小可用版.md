# 标准化字段字典与入库判定规则（最小可用版）

## 1. 目标
将不同平台、不同格式（CSV/JSON/Markdown/纯文本/AI 抽取）的原始数据，统一映射为同一套 `video` 标准字段，并按质量分进行分层入库。

## 2. 标准字段字典（video）

| 字段名 | 类型 | 说明 | 典型来源别名 |
|---|---|---|---|
| `id` | BIGINT | 系统视频主键（哈希生成） | `id/video_id/aid/aweme_id/bvid` |
| `dedupe_key` | VARCHAR(128) | 去重键（平台+URL 或 平台+标题+作者） | 系统生成 |
| `platform_video_id` | VARCHAR(128) | 平台原始视频 ID | `video_id/id/aid/aweme_id/bvid/bv` |
| `title` | VARCHAR(255) | 视频标题 | `title/video_title/name/标题` |
| `author` | VARCHAR(100) | 作者/UP 主 | `author/up/uploader/博主` |
| `source_platform` | VARCHAR(32) | 来源平台标准值 | `platform/source/平台/来源平台` |
| `source_url` | VARCHAR(1000) | 来源链接（去 query/hash） | `url/link/video_url` |
| `category` | VARCHAR(60) | 分类（统一为中文常见类目） | `category/type/tname/分类/分区` |
| `play_count` | BIGINT | 播放量 | `play/views/view_count/播放量` |
| `like_count` | BIGINT | 点赞量 | `like/likes/点赞量` |

| `share_count` | BIGINT | 分享量 | `share/shares/转发量` |
| `favorite_count` | BIGINT | 收藏量 | `favorite/collect/收藏量` |
| `duration_sec` | BIGINT | 时长（秒） | `duration/length/时长` |
| `publish_time` | DATETIME | 发布时间 | `publish_time/pubdate/date/time/发布时间` |
| `import_type` | VARCHAR(32) | 导入方式 | `text_import/file_import/mock/url_crawl` |
| `source_file` | VARCHAR(260) | 来源文件名 | 上传文件名 |
| `import_time` | DATETIME | 导入时间 | 系统生成 |

| `import_job_id` | BIGINT | 所属导入任务 ID | 系统生成 |

## 4. 入库判定规则（最小可用）

### 4.1 质量评分
系统按标题、作者、平台、分类、互动指标、来源链接、时长、标签、AI 置信度等打分，并对明显异常（如点赞>播放）扣分，最终得到 `data_quality_score`。

### 4.2 质量分层
- `GOOD`：`score >= 60`
- `LOW`：`40 <= score < 60`
- `REJECTED`：`score < 40`

### 4.3 入库策略
- `GOOD/LOW`：写入 `video` 主表。
- `REJECTED`：不写入 `video`，写入 `import_reject_record`，保存：
  - 拒绝原因 `reject_reason`
  - 修复建议 `suggest_fix`
  - 原始片段 `raw_excerpt`
  - 质量分 `quality_score`
  - AI 信息 `ai_used/ai_confidence`

## 5. 去重规则
- 主规则：`tenant_user_id + dedupe_key` 唯一。
- `dedupe_key` 生成逻辑：
  - 有 URL：`sha1(platform + '|' + normalized_url)`
  - 无 URL：`sha1(platform + '|' + normalized_title + '|' + normalized_author)`

## 6. 导入结果指标
导入接口返回：
- `records`：解析总条数
- `acceptedRows`：通过规则条数（GOOD+LOW）
- `lowQualityRows`：低质量但入库条数
- `rejectedRows`：拒绝条数
- `newVideos/updatedVideos`：新增/更新视频数
- `users/behaviors`：用户与行为写入数

## 7. 前端可见能力
数据管理页支持：
- 来源追踪：查看最近入库记录
- 低质量/拒绝记录：查看拒绝原因与修复建议，指导用户修正后重导
