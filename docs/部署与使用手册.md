# 部署与使用手册

## 1. 环境要求
- MySQL 8.x（已启动）
- Python 3.10+
- JDK 17+
- Maven 3.9+
- Node.js 18+

数据库默认参数：
- Host：`localhost`
- Port：`3306`
- User：`root`
- Password：`你的MySQL密码`
- DB：`video_analysis`

## 2. 一键部署（推荐）
在项目根目录双击：
- `OneClick_Full_Auto_Launch.bat`

它会自动执行：
1. Python 依赖安装
2. 数据库初始化（无需 MySQL CLI）
3. 启动后端与前端
4. 运行自测并输出报告
5. 打开浏览器

## 3. 手动部署
### 3.1 初始化数据库

## 4. 登录与账号体系
- 首页为登录/注册页（非直接业务页）
- 默认测试账号：`demo`
- 默认测试密码：`123456`

说明：
- 平台已实现按账号的数据隔离。
- 不同账号登录后看到的是各自的数据空间。
- 新注册账号默认无历史数据，可通过“数据采集”模块导入自己的数据。

## 5. 数据采集与入库
页面入口：`http://localhost:5173/crawler`

支持：
- URL 采集（平台可选）
- 文本导入
- 文件导入：`.md` `.markdown` `.txt` `.log` `.csv`

导入后会自动写入当前登录账号的数据空间。

## 7. 常见问题
1. 登录接口 404：
- 原因：后端仍是旧进程。
- 处理：关闭旧后端后重新 `mvn spring-boot:run`。

2. 前端能打开但数据接口 401：
- 原因：未登录或登录已过期。
- 处理：返回首页重新登录。

3. 初始化时报列不存在：
- 处理：使用最新 `scripts/init_db_python.py`，它会自动补齐旧库字段。
